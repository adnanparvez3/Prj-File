ABSTRACT

The React-e-plantShopping project, also known as Ziads Plants, is a premium full-stack e-commerce platform designed to bridge the gap between plant nurseries and urban gardeners. Built using the MERN (MongoDB, Express, React, Node.js) stack, the application provides a sophisticated, glassmorphic user interface that offers an immersive shopping experience. The system facilitates seamless plant discovery through categorized grids, advanced shopping cart logic using Redux Toolkit, and a robust Cash on Delivery checkout system. Furthermore, it incorporates a comprehensive Admin Dashboard that empowers nursery owners with real-time inventory control and order tracking capabilities. By integrating modern web technologies like Framer Motion for animations and Lucide-React for iconography, the project delivers a state-of-the-art digital storefront that manages everything from plant seeding in the database to final order fulfillment and stock synchronization.

With the rise of the MERN stack as a standard for high-performance web applications, there is an opportunity to create lightweight yet powerful e-commerce solutions that can be deployed easily on the cloud with MongoDB Atlas and Node.js.

4.1 PROBLEM DESCRIPTION
The core challenge addressed is the synchronization between the front-end shopping experience and the back-end inventory levels. Specifically, ensuring that when a customer completes a Cash on Delivery (COD) order, the stock count in the MongoDB database is accurately decremented, and the admin is immediately notified via the dashboard.

4.2 THE CATALOG AND CART MODULE
This module handles the display of various plant categories. Using Redux Toolkit, the application maintains a global cart state. Features include:
- Category Filtering: Users can browse plants based on their utility (Aromatic, Medicinal, etc.).
- Cart Constraints: The system checks the 'stock' attribute from the database to prevent users from adding more items than available.
- Price Parsing: A robust logic handles costs formatted as strings (e.g., $15) and converts them to integers for subtotal calculations.

4.3 ORDER FULFILLMENT AND ADMIN MODULE
This module manages the transition from cart checkout to order recording.
- COD Shipping Form: Captures Name, Email, Phone, and Address.
- Inventory Auto-Decrement: A post-order trigger in the Express API reduces stock for each plant ordered.
- Admin Reporting: A dedicated view in the dashboard lists all orders with date stamps, customer info, and total revenue calculations.

CHAPTER-6

6.1 PROBLEM DISCUSSION
One of the primary discussions in this implementation is the handling of stock concurrency. If two users attempt to buy the last unit of a plant simultaneously, the system must ensure the order is granted to the first successful requester while blocking the second. This is handled by verifying stock levels both in the frontend UI and during the order save process in the backend.

6.2 IMPLEMENTATION OF PROBLEM
The solution is implemented using a Mongoose schema that includes a 'stock' field. In the Express POST /api/orders route, a loop iterates through the requested items and performs a $inc: { stock: -item.quantity } operation on the Plant collection. This ensures database-level consistency during high-traffic scenarios.

9.1 CODING

// RELEVANT CODE SNIPPET: EXPRESS API FOR ORDER PROCESSING
// File: server/index.js

app.post('/api/orders', async (req, res) => {
  try {
    const orderData = {
      username: req.body.username,
      email: req.body.email,
      phone: req.body.phone,
      location: req.body.location,
      items: req.body.items,
      totalAmount: req.body.totalAmount
    };

    const order = new PlantOrder(orderData);
    const savedOrder = await order.save();

    // Decrement stock for each item in the order
    for (const item of req.body.items) {
      if (item.plantId && mongoose.Types.ObjectId.isValid(item.plantId)) {
        await Plant.findByIdAndUpdate(
          item.plantId,
          { $inc: { stock: -item.quantity } }
        );
      }
    }
    res.status(201).json(savedOrder);
  } catch (err) {
    res.status(400).json({ message: err.message });
  }
});

// RELEVANT CODE SNIPPET: REDUX TOOLKIT FOR CART LOGIC
// File: src/CreateSlice.jsx

export const CreatSlice = createSlice({
  name: 'cart',
  initialState: { items: [] },
  reducers: {
    addItem: (state, action) => {
      const item = action.payload;
      const existingItem = state.items.find((i) => i.name === item.name);
      if (existingItem) {
        if (existingItem.quantity < existingItem.stock) {
          existingItem.quantity++;
        }
      } else {
        state.items.push({ ...item, quantity: 1 });
      }
    },
    removeItem: (state, action) => {
      state.items = state.items.filter(i => i.name !== action.payload.name);
    },
    updateQuantity: (state, action) => {
      const { name, quantity } = action.payload;
      const item = state.items.find(i => i.name === name);
      if (item) {
        item.quantity = quantity;
      }
    }
  }
});
